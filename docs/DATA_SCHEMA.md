# 資料結構：食材資料

## 分類架構
四大分類：蔬菜、水果、海鮮、肉品（可再細分次分類，例如海鮮下分魚類／甲殼類／貝類／頭足類）。

## 基本欄位（每筆食材）

| 欄位 | 說明 | 備註 |
|---|---|---|
| name | 中文名／俗名 | 市場常用俗名優先，例如「透抽」 |
| category / subcategory | 四大分類 → 次分類 | |
| image | 圖片 | 建議整體＋切面各一張 |
| months / seasonNote | 產季 | 蔬果海鮮用月份陣列；肉品多半全年供應，用文字註記即可，不必勉強套用月曆 |
| characteristics | 外觀特性、口感 | 2–3 句 |
| selectionTips | 挑選技巧 | 3–5 條，精簡條列，不寫長文 |
| nutrition | 營養價值 | 3–4 個重點數值即可，不需列全表 |
| storage | 保存方式 | |
| verification | 各欄位的查證狀態 | 分級定義見下方「資料驗證分級」 |

## 品種（variants）處理邏輯
不是每個食材都要拆品種，**只有品種間「產季」或「挑選技巧」有明顯差異時才拆**（例如芒果、葡萄、地瓜、茶）。若品種只是外觀細節差異、選購邏輯相同，用一句「常見品種：OO、OO」帶過即可。

共用資訊（分類、營養、保存方式）放在食材主體層級，不隨品種變動；產季、外觀、挑選技巧則依品種切換。

```json
{
  "name": "芒果",
  "category": "水果",
  "nutrition": [ /* 共用 */ ],
  "storage": "...",           // 共用
  "variants": [
    { "variant_name": "愛文", "months": [5,6,7,8], "appearance": "...", "selectionTips": ["..."] },
    { "variant_name": "金煌", "months": [6,7,8],   "appearance": "...", "selectionTips": ["..."] }
  ]
}
```

UI 上先用簡單的品種列表／pill 切換即可，不用一開始就做複雜元件。

## 肉品的欄位彈性
肉品沒有現成的公開資料庫，且多為全年供應：
- 產季欄位：直接標「全年供應」或隱藏，不用勉強套產季月曆
- 挑選技巧：先用一組**通用原則**（顏色、油花、彈性、包裝標示、有效期限）套用在整個肉類分類即可，不用每個品項客製化

> 原則：欄位是「有就顯示、沒有就用通用內容或隱藏」的彈性欄位，不是每筆資料都要填滿同一套欄位。

## 內容資料來源
- 營養資料：衛福部食藥署「食品營養成分資料庫」，政府資料開放平台有開放資料集與 API（data.gov.tw/dataset/8543）
- 蔬果產季／產地：農業部資料開放平台（data.coa.gov.tw）、農糧署「農產品產地產期查詢」「水果月曆」
- 海鮮產季：無完整官方資料，需搭配民間整理來源（如食令日曆、天下雜誌台灣四季海鮮指南）交叉驗證
- 肉品：無公開資料庫，挑選原則需自行整理

## 資料驗證分級

食材資料多半先由 AI 協助草擬，品質參差不齊，需要一套機制標示「這筆內容有沒有被查證過、查到什麼程度」。每筆食材的 `nutrition`、`selectionTips`、`months` 三個欄位**各自獨立**記錄驗證狀態（因為同一筆食材，營養可能已查證、產季卻還沒），格式為：

```json
"verification": {
  "nutrition": { "level": "official", "source": "..." },
  "selectionTips": { "level": "secondary", "source": "..." },
  "months": { "level": "unverified" }
}
```

### 三個等級的定義

| level | 定義 | 適用欄位 |
|---|---|---|
| `official` | 直接比對政府開放資料 / 官方資料庫的結構化資料，數值一致或已依官方數據修正 | 主要用於 `nutrition`（食藥署食品營養成分資料庫有 API 可查） |
| `secondary` | 找到**至少兩個彼此獨立**的可靠來源（政府機關網站、農會／漁會、主流媒體、專業健康媒體）說法一致；若多篇文章其實互相轉載同一篇原始報導，不算獨立來源 | 主要用於 `selectionTips`（挑選技巧散落在網路文章，沒有結構化 API 可查，只能用交叉比對的方式提高信心） |
| `unverified` | 尚未查證，內容為 AI 草擬或既有知識估算，未對照任何外部來源 | 預設狀態；`months`（產季）目前刻意全部保留在此等級，由使用者親自對照農糧署資料人工核對，不透過網路搜尋自動升級 |

### 查證時的原則
- 品種內任一 variant 缺乏獨立佐證時，整筆（含其他 variant）保守保留在較低等級，不因部分品種查到資料就整體升級
- 查證過程中若發現原本內容與可靠來源有出入（例如判斷部位、量測數值錯誤），應直接修正內容本身，而不是保留錯誤內容卻標記已驗證
- `source` 欄位需具體註明查到的來源（機構名稱、網站、樣本編號等），方便日後回頭複查
