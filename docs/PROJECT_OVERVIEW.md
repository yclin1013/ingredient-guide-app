# 專案概觀：食材圖鑑 App

## 目的與定位
收錄台灣常見蔬菜、水果、海鮮、肉品，協助使用者在傳統市場快速了解「當季嗎、怎麼挑」。

- 前期定位：個人興趣專案，同時作為作品集 (portfolio)
- 未來視情況：評估是否發展為正式產品或商業化（例如與在地小農合作、導購等）
- 料理相關功能：明確排除在目前規劃之外，列入未來階段

## 目標使用者與核心情境
逛傳統市場時，站在攤位前，想快速知道：
1. 這個食材現在是不是當季？
2. 怎麼挑選品質好的？

因此整體設計以「手機上快速查閱」為核心，而非長篇閱讀。

## 資訊架構
```
首頁（本月當季精選 / 分類入口 / 搜尋）
  → 分類瀏覽（蔬菜 / 水果 / 海鮮 / 肉品）
      → 食材列表（可依產季篩選）
          → 食材詳細頁
```

食材詳細頁包含：圖片、產季、外觀特性、挑選技巧、營養價值、保存方式。
（欄位細節與資料結構見 `DATA_SCHEMA.md`）

## 技術棧（初步方向，最終待確認）
- App：Flutter 或 React Native（跨平台，適合個人開發＋作品集展示）
- 後端／資料：Firebase 或 Supabase（BaaS，MVP 規模不需要自建後端）
