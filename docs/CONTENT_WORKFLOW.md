# 食材資料批次新增流程

> 用途：使用者說「照流程新增 N 種某分類」時，依本文件執行，不需要每次重新討論規則。
> 欄位定義與驗證分級的完整規範見 `DATA_SCHEMA.md`，本文件只講操作流程。

## 1. 品項挑選原則（依分類）

| 分類 | 挑選標準 |
|---|---|
| 蔬菜 | 台灣本產、傳統市場常見、一般家庭常買，不選罕見或進口為主品項 |
| 水果 | 同上：台灣本產、傳統市場常見，不選罕見或進口為主品項 |
| 海鮮 | 台灣近海捕撈或養殖常見品項，不選罕見或以進口為主的海產 |
| 肉品 | 以常見「部位」為單位（例如豬五花、雞腿），不是品種或品牌；不需要像蔬果一樣講究產地 |

## 2. 避免重複

新增前，先讀取該分類現有的資料檔（`mobile/src/data/{分類}.json`），列出已收錄品項的 `id` 與 `name`，新增的品項不可與其重複。

## 3. 確認閘門

品項清單選好後，**先列給使用者確認，不要直接開始搜尋或寫入**。使用者確認後才進行後續的資料搜尋與驗證流程。

## 4. 驗證分級規則

三級制（`official` / `secondary` / `unverified`）定義見 `DATA_SCHEMA.md` 的「資料驗證分級」章節。以下是每個欄位在批次新增流程中的具體查證方式：

### 4.1 營養欄位（nutrition）

透過衛福部食藥署「食品營養成分資料庫」API（data.gov.tw/dataset/8543）逐筆核對，成功比對到官方樣本才標記 `official`，並在 `source` 註明樣本名稱與整合編號。

**海鮮的特殊情況**：該資料庫將海鮮歸類在「魚貝類」，樣本用的是官方／學名（例如「台灣鎖管」對應市場俗稱「透抽」），與市場俗名常常對不上，直接查詢俗名找不到資料是正常現象。遇到這種情況時：

1. **不可以查不到就用網路搜尋的營養數據替代**——營養數值只能來自食藥署資料庫，這是唯一來源。
2. 改用「名稱對應解析」機制（見第 5 節）先找出可能對應的官方名稱／學名，再用候選名稱重新查詢 API。
3. 對應後仍查不到，或名稱對應本身無法確定，該筆營養欄位保留 `unverified`，並在完成後的回報中列出，不可以为了让欄位好看而編造或估算數字。

### 4.2 挑選技巧（selectionTips）

上網搜尋佐證來源，需要**至少兩個彼此獨立**的可靠來源（政府機關網站、農會／漁會、主流媒體、專業健康媒體）說法一致才標記 `secondary`。判斷「獨立」時要注意：多篇文章如果只是互相轉載同一篇原始報導（例如都引用同一個原始出處、內容幾乎逐字相同），不算獨立來源。

查證過程中若發現原有內容與可靠來源有出入，直接修正內容本身，不要保留錯誤內容卻標記已驗證。

品種（variants）之間，若其中任一品種缺乏獨立佐證，整筆（含其他品種）保守保留在較低等級，不因部分品種查到資料就整體升級。

### 4.3 產季（months / seasonNote）

**一律保留 `unverified`，不透過網路搜尋自動升級**，等使用者親自核對。人工核對時，依分類參考不同來源：

- 蔬菜、水果：農糧署「農產品產地產期查詢」、「水果月曆」
- 海鮮：沒有完整官方資料，改用民間整理資料（如食令日曆、天下雜誌台灣四季海鮮指南）交叉驗證，或請教市場攤商的實地經驗

### 4.4 肉品的例外規則

肉品的產季與挑選技巧不走上述查證流程：

- **產季**：直接標「全年供應」（`seasonNote: "全年供應"`），不需要驗證，`verification.months` 可直接視為不適用這套查證邏輯（沒有產季可查）
- **挑選技巧**：沿用一組通用原則（顏色、油花、彈性、包裝標示、有效期限）套用在整個肉類分類即可，不用每個部位客製化，也因此不需要像蔬果海鮮那樣逐筆找獨立來源佐證

## 5. 海鮮的「名稱對應解析」機制

海鮮用市場俗名查食藥署資料庫失敗時，依以下步驟處理，**不可以直接放棄或用網路數據代替官方數值**：

1. **搜尋正式名稱／學名**：上網搜尋該市場俗名對應的正式中文名與學名，優先參考漁業署、水試所（台灣水產試驗所）、魚類資料庫等可靠來源。
2. **列出候選名稱**：搜尋結果可能對應到不只一個名稱或物種，把候選都列出來。
3. **逐一用候選名稱查詢 API**：拿每個候選名稱重新查詢食藥署資料庫，找有沒有比對得上的樣本。
4. **成功對應時**：
   - 在該筆資料新增 `aliases` 欄位，記錄「市場俗名、官方名稱、學名」三者對應關係，方便之後重複查詢直接沿用，不用每次重新搜尋。
   - 在 `verification.nutrition.source` 註明是依據名稱對應查到的，並簡述對應依據（例如：「經查證，市場俗名『透抽』對應學名 Uroteuthis chinensis，即食藥署資料庫樣品『台灣鎖管』」）。
5. **名稱對應以學名為準，不以俗名相似度判斷**——台灣海產常見「一名多種」或「一種多名」的情況（例如透抽／小卷／中卷其實是同一物種依大小分級的俗名；但也存在不同俗名字面相似、實際是不同物種的情況）。判斷對應是否成立，要看學名是否一致，不能只看俗名唸起來像不像。
6. **仍有多個物種可能時，不可自行挑一個**：如果搜尋後發現這個俗名在市場上實際對應多個不同物種（俗名混用），必須標記該筆為「名稱待確認」，該筆營養欄位保留 `unverified`，並在回報時列出各候選物種與其差異，交由使用者決定要採用哪一個。

## 6. variants 使用時機

品種間「產季」或「挑選技巧」有明顯差異時才拆 variants，多半發生在水果（例如芒果的愛文／金煌／玉文）。海鮮、肉品通常不需要使用 variants 結構。

## 7. 完成後回報格式

全部品項處理完後，回報兩個部分：

1. **每筆資料的驗證狀態總覽**：一張表列出每個品項在 `nutrition` / `selectionTips` / `months` 三個欄位各自的驗證等級。
2. **建議優先人工查證的項目**：哪些品項驗證狀態偏弱（例如整筆都是 unverified、或名稱對應不確定被標記「待確認」），列出來提醒使用者優先處理。
